GA technologiesでソフトウェアエンジニアをしている中坂です。

今回は日々の業務におけるPersonal Knowledge Management（PKM）について、自分が普段実践している方法を書いてみます。最近はLLMと組み合わせることで通常業務でも活躍しており、自分にとってはすでに第二の脳として十分機能していることを実感しています。

## 前提

もう気づけばアラフォーなのでワーキングメモリが日に日に小さくなっている感覚があります。思いついたアイデアや「後でやろう」と思っていたことはすぐに揮発してしまいます。MTGで話した内容も翌日には半分くらい曖昧になっており非常に悲しいです。

そこで今は基本的に自分の記憶力というものを信じないことにしており、代わりに乱雑でいいからとにかく何でもテキストにするようにしています。仕事の中で発生する思いつき、雑なメモ、MTGや何気ない会話で出てきたこと等々をとにかくメモし、一旦全て脳内から追い出して外部記憶に保持する戦略をとっています。

## Obsidian

ツールとしては [Obsidian](https://obsidian.md/) を使っています。Obsidian自体は2020年頃から開発が進んでおり、LLMが広まっていく中で再ブレークしたMarkdownベースのノート作成アプリです。<iframe src="https://embed.zenn.studio/card#zenn-embedded__ea9f0b8ededf5" frameborder="0" height="121.328125"></iframe>

Obsidianを選んだ理由は、 **ローカルファイルベースのMarkdownエディタであること** 、 **コミュニティがそれなりに活発であること** くらいで、特に強いこだわりがあるわけではないです。実はObsidianの機能は詳しく知らず、ただのMarkdownエディタとしての使い方しかできていません。もっとしっかりとPKMを実践している方々と比べると全然使いこなせていないでしょう。綺麗に使うよりも毎日雑にでも使い続けることを優先しています。

綺麗に運用しようとしてもどうせ続かないしそもそもルールを忘れます。変に固定化したルールを決めるとそれが破られた時にモチベーションが落ちてしまうといったこともおきます。なので **うまくやらないというこだわりだけは強く持つ** ようにしています。

ちなみに現在のファイル数を確認するために `find . -name "*.md" | wc -l` を実行してみたところ1362ファイルあることがわかりました。業務時間ベースで大体1日に4ファイルくらい作成しているようです。

## メインの使い方

自分のObsidian運用で最も中心的なのが **毎日「dailyページ」を作成すること** です([Logseq](https://logseq.com/) などのツールで言うところのjournal)。その日専用の自分だけのホワイトボードを用意するイメージです。

![](https://res.cloudinary.com/zenn/image/fetch/s--liUpkIpZ--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_1200/https://storage.googleapis.com/zenn-user-upload/deployed-images/edb9f882d2440bb578cff565.png%3Fsha%3D5c7543669eb39529048584f55d01e965c0d81bdc)

悲しいことに一目で一覧できる範囲のことしか考えられないようになってしまったので、その日に関することはこの一枚に全部書くようにしています。

## 始業時にやること

[obsidian-calendar-plugin](https://github.com/liamcain/obsidian-calendar-plugin) というプラグインを入れているので、常にObsidianのサイドバーにカレンダーが表示されています。

<iframe src="https://embed.zenn.studio/card#zenn-embedded__660e655002434" frameborder="0" height="121.328125"></iframe>

毎朝仕事を始めたら最初にこのCalendarの今日の日付をクリックします。すると `daily/20260221.md` のような形式の空のdailyページが作成されます。

ここで重要なのが **前日の内容をコピペして今日作成した空のdailyページに貼り付けること** です。これが意外と大事で、前日の内容を一旦コピペして引き継いでから今日分として編集していくことで、TODOのやり残しの漏れや昨日やったことなどを思い出しながら記載できます。

## ページ構成

dailyページには以下のようなカテゴリを並べています。

- **予定**
- **今日やること**
- **タスクキュー**
- **メモ**
- **日報**

まずGoogle Calendarを見ながらMTG等の作業時間以外の予定を確認し、「 **予定** 」にリスト形式で雑に書いていきます。例えばこんな感じです。

```
## 予定
- 11:00 デイリースクラム
- 13:30 oooに関する打ち合わせ
- 18:00 [[1on1/AI太郎]]
```

ちなみにObsidianでは `[[]]` で囲うとそのまま新規ページまたは既存ページへのリンクになります。1on1のような場合はメンバー名でリンクを作っておくと、個別ページが作成されるので後から参照しやすくなります。

<iframe src="https://embed.zenn.studio/card#zenn-embedded__604689d9af42a" frameborder="0" height="121.328125"></iframe>

「 **今日やること** 」には今日やらないといけないことや前日やり残したことを書いておきます。

```
## 今日やること
- PRのレビュー
- 問い合わせ対応
- OOOのissueの実装
- XXXのissueの実装
- 資料確認
```

「 **タスクキュー** 」・「 **メンバーリスト** 」・「 **メモ** 」・「 **日報** 」は朝の段階では特に何か書いたりしませんが、サクッと目を通します。

特に「 **タスクキュー** 」にはいつかやらないといけないことや、考えなければいけないことが雑に積まれています。

```
## タスクキュー
- OOOの技術選定についてまとめる
- XXXのリファクタリング方針を考える
- モバイル開発でAIを使った場合と使わなかった場合でアウトプットの差がどの程度あるか調べる
- OOOプロジェクトの課題設定をもう少し良い感じにできないか
- 来月の勉強会のテーマを決める
- [[採用/面談テンプレ]] 面談のフローを見直す
- KPIの見直し
```

**毎朝ここをチラ見しておくと常に脳内の隅っこに存在し続けるので、暇な時にふと思い出したりして、良い考えが浮かんだします** 。また、「 **タスクキュー** 」の中から一部を「 **今日やること** 」に移動して「やるぞ！！」という意識を強くしたりすることもあります（効果は不明）。

あとは完了済みのタスクやもう不要だなと思ったものは削除していきます。

大体朝5分〜10分くらいでこれだけやります。

今はこれらは仕事を始める前にエンジンをかけるための儀式になっています。

## 仕事中の使い方

仕事中は基本的にこのdailyページを開いておきます。

## メモへの箇条書き

MTGや1on1などで出てきた話、ふとした時に思いついたことは「メモ」に箇条書きしていきます。ここに書く内容は本当に雑で構いません。後で読み返して何とか意味がわかる程度で十分です。

## \[\[リンク\]\]で気軽に新規ページを作る

![](https://res.cloudinary.com/zenn/image/fetch/s--mwx5U6ji--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_1200/https://storage.googleapis.com/zenn-user-upload/deployed-images/ac5e845d30e1a3dd3a8d7c9e.png%3Fsha%3Db8a0b745c161cd45e9ab8026723fffbbabbe3ec3)

メモに書くにはあまりに長くなりそうな話題や内容が出てきた時には、 `[[OOOについて]]` みたいな感じで新規ページへのリンクを作成し、その中に書いていきます。こういう運用をしていると乱雑にページが増えるので、Vault（Obsidianのメモディレクトリ）は基本ぐちゃぐちゃです。

一応デフォルトで新規ページが作られるディレクトリを設定することはできるので今は `雑/` という場所を作ってそこに自動で作成されるようにはしていますが、何かを探すときには基本的に全文検索を使うのでどこにファイルがあろうと関係ないです。

Obsidianにはカテゴリやタグをつける機能もあるのですが、自分は記事同士をグラフで繋いて可視化したりといったこともしないので特に困ってはいないです（以下は現在のグラフビュー。何も有益な情報がない。）

![](https://res.cloudinary.com/zenn/image/fetch/s--IAJm8sll--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_1200/https://storage.googleapis.com/zenn-user-upload/deployed-images/dd52e096f661289943117523.png%3Fsha%3Dd07009211795836d2019104ab5281ad05b543920)

ただ、実際この辺りの整理をうまくやれるとよりPKMが楽しくなり有益なのだろうとは思いますが自分には出来なかったです…

## タスクキューへの積み方

もしMTGなどでNext Actionが設定されたり、なんとなく考えておかないといけないようなことが湧いてきたら「 **タスクキュー** 」に雑に積みまくります。そして明日以降の自分が頑張ってくれることを祈ります。

## 仕事後の使い方

「 **日報** 」に1日の振り返りを書きます。

一日仕事をしていると「 **メモ** 」にMTGで出た話や思いついたことなど何かしらの文章が書かれているはずです。

```
## メモ
- OOOの障害対応について
  - 初動対応でオンコール体制がちゃんと機能していただろうか
  - 連絡フローの見直しは必要かも
- コードレビュー。コードのコメントが少ない。可読性はあっても意図がわからないケースが多いのでは。
- 技術スタックとアーキテクチャパターンの話。CQRSについてもう少し学んだほうがいいかもしれない。
```

それをネタにして、なんとなく考えたことを「 **日報** 」に書きます。

```
## 日報
- コードレビュー
  - AIが生成したコードのレビューをどうすべきかという話になった。コード単体の可読性を見るのか、振る舞いの検証で良しとするのか。
  - 人間がコードを保守する前提なら従来のレビュー観点が必要だが、生成も保守もAIがやる世界だとレビューの意味が変わってきそう。
- 昼飯の時の会話
  - RoR以外の技術スタックについての話を扱った。個人的にはフロントエンド開発が面倒という感覚が正直なくて、Railsに固執する気持ちがあまりない。
  - デプロイ前にビルドしたら落ちてくれるようにしたいので静的型の言語を使いたいという気持ちはあるよねという話が出て、それなーと思った。
```

内容は適当で良いです。最初のうちなら「昼に食ったガパオ弁当が旨くてやっほー」とかでも問題ありません。 **自分の内面を言語化する習慣を作る訓練なので、まずはとにかく自分に向き合い、言葉にすることを続けるのが大事** 、という気持ちでやっています。

## その他の活用

## Kindle Highlightsの同期

[Obsidian Kindle Plugin](https://github.com/hadynz/obsidian-kindle-plugin) を使って、過去に読んだKindle本のハイライトを同期しています。

<iframe src="https://embed.zenn.studio/card#zenn-embedded__8192930fb4f97" frameborder="0"></iframe>

冒頭にも書いた通り記憶力が衰えているため、本を読んでも大体忘れます。忘れてしまうなら引き出せるように外出ししておき、忘れても何とかして探し出せるようにしておけば良いです。

![](https://res.cloudinary.com/zenn/image/fetch/s--hh_qA526--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_1200/https://storage.googleapis.com/zenn-user-upload/deployed-images/cf3bf41c621402eec02bab4a.png%3Fsha%3Da53dd0df82f9e66fe8775bb2cb72e4b0c64c6db4)

この機能を使うと「なんかこんなこと書いている本があったよな〜」と思った時に全文検索やLLMに読ませることで雑に掘り起こせたりするので、一気に第二の脳感が出てきて便利です。

## 1on1ディレクトリ

日頃いろいろな人と話をする中で、心に留めておきたいフィードバックを得たり、何かしら気にかけておくべきことを得られたりすることがあります。

いわゆる1on1などで話をした場合は、普通のメモとは別にその人との会話であることも重要な情報なので記録しておくと便利です。過去にどういう話をしたのか思い出せるようにしておくと、脳をすぐにスイッチできて有意義な時間にしやすくなります。

先ほども書いたように `[[1on1/AI太郎]]` や `[[1on1/LLMの助]]` みたいな感じで気軽にページを作れるので私のObsidianには `1on1/人物名.md` のファイルがたくさんあります。

もしマネージャーなどの管理者であれば、 **日々の育成や評価の際に使える重要な情報源** にもなるので便利です。

## LLMとの連携

日々の考えなどをメモしまくっていると、何か資料を作りたい時にひとまずこのメモ全体を参照させて何かしらアウトプットを出してみるということができます。 **何事も最初の一歩目が重くて大変なので、完成度が10%でも20%でもいいから何か叩き台があると作業の捗り方が全然違います** 。

例えば自分は以下のような使い方をしています。

- メモから資料の叩き台を生成する
- 技術的な調査メモをもとに会社ブログのネタを考えてもらう
- 1週間のdailyページや雑なメモを元に週報を作成してもらう
- 1on1メモから振り返りやフィードバックのポイントを整理する

ただしLLMを活用するためにはとにかくデータが必要です。データがなければ何も始まりません。 **仕事に関して見聞きしたこと、自分の脳内に湧いてきたものは雑に書き出しておくという習慣が重要** です。何か形にしておけばLLMであとは何とかなります。

## 永続化

これは人によって色々なやり方があると思いますが、自分の場合は社外に公開してはいけない話も含まれるので会社の規定に沿った方法でバックアップをとるようにしています。Obsidian自体はローカルファイルベースなので、突然PCが壊れたりすると詰みます。定期的なバックアップは忘れないようにしましょう（自戒）。

## まとめ

仕事でObsidianをどのように使っているかを書きました。

自分の場合は役職的にMTGひとつとっても様々な種類があり、コミュニケーションする相手も幅広いです。そういう状況だと、そこで話した内容全てをワーキングメモリに載せ続けておくことなんてまず無理です。なのでまずは全部雑に書き出して脳内から消して外部記憶に保持します。dailyページを中心に仕事に関して考えておくべきことを常に目に見える範囲に広げておく戦略で、老いに抗いながらなんとかやっています。

あとは日々の考えや技術的な調査メモ等も全部まるっと雑にテキスト情報として吐き出しておくのは、時間が経てば経つほど効いてきます。今の時代はデータさえあればいざという時にLLMでいい感じに活用できるようになるので、雑でもいいからとにかく書き続けることの価値はどんどん上がっていると感じています。

CalendarプラグインやKindle Highlightsプラグインなど一部使っているものはありますが、Obsidianに実装されている機能はほとんど知らないしカスタマイズもあまりしていません。特別なハックをしたり綺麗に使うことよりも毎日雑にでもいいから普通に使い続けることのほうがずっと大事だと思っています。
